角色：你是“北港机器翻译中心”的模型负责人，直接对交付质量与训练预算负责。

世界：时间是季度冲刺前夜。团队长期使用 encoder-decoder 的 recurrent/convolutional 方案，并通过 attention 连接。你收到一份新方案：a new Transformer, based solely on attention。

旧系统还能跑，但竞品在 WMT 榜单上持续拉开差距。你必须在稳定、成本与变革之间做出可落地的选择。

情报卡

本关目标：在三轮决策中完成架构定案、训练策略与任务扩展路径，平衡稳定(stability)、预算(treasury)、改革(reform)。

  • 情报卡A：来源明确对比了 dominant models 与 new Transformer 的结构差异，关键词是 based、attention、transformer。
  • 情报卡B：来源给出 translation 结果：WMT English-to-German 28.4 BLEU、English-to-French 41.8 BLEU，并强调 best results 与训练成本对比。
  • 情报卡C：来源指出模型更可并行且训练时间更少，但未提供你当前集群的完整迁移损耗；相关评估需保持保守措辞。
  • 情报卡D：来源 show 了对 other tasks 的可迁移性（English constituency parsing），但没有覆盖你全部业务场景。

    模块一：替换引擎的72小时

    你推门进入会议室，导师把一句话写在白板上：如果 attention 足够，是否还需要 recurrence 与 convolutions？团队沉默，倒计时开始。

    主线任务：在不虚构数据的前提下，依据已知结论推动架构决策：是否从旧范式迁移到 Transformer，并在 translation 与其他 tasks 上制定推进节奏。

    导师：林岚（平台总监）——“只用来源里能站得住的结论，不夸大，不保守拖延。”

    你完成了从“旧范式惯性”到“attention-only 决策框架”的过渡：先定架构，再控训练，再测泛化。每一步都以可核验结论为锚，不拿未知信息冒进。

    当证据已明确指向 quality + parallelization + training efficiency 时，正确策略不是盲目激进，而是基于已知边界推进 Transformer 的分阶段落地。
